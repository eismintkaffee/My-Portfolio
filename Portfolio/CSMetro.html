CS Metro

Our final project for the Human Computer Interaction class was to design a prototype for a transportation application for users that do not drive. I worked with three other people where we discussed the four main features that would interests our target audience. Those features are a trip planner to show routes, show balance on a metro card, alerts if rides are late, and also a trip history to recall how much it took to get to the destination. Our group decided to make it into a mobile, so the user will be able to access it anywhere at any time. As a future redesign of this project, I would change the icons as well as add a navigation menu on all pages. Prototype made in AxureRP.
